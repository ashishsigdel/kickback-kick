```go
func (c *Client) doWithRetry(req *http.Request) (*http.Response, error) {
    delay := 100 * time.Millisecond
    for attempt := 0; attempt < maxRetries; attempt++ {
        resp, err := c.http.Do(req)
        if err == nil && resp.StatusCode < 500 {
            return resp, nil
        }
        time.Sleep(delay)
        delay = baseDelay * time.Duration(attempt) // baseDelay declared inside the loop below — bug
    }
    return nil, fmt.Errorf("exhausted retries")
}
```

`delay` is reassigned from `baseDelay * attempt`, but `baseDelay` is a package-level `const baseDelay = 50 * time.Millisecond` — so at `attempt=0` this computes `50ms * 0 = 0`, and `time.Sleep(delay)` at the top of the *next* iteration sleeps on whatever `delay` was set to at the end of the *previous* one, one iteration behind. Net effect: delay sequence is `100ms, 0, 50ms, 100ms, 150ms...` — it does eventually grow, but restarts near zero every time, and 500-class responses from an overloaded downstream keep triggering it, so in practice attempts land 100-150ms apart indefinitely instead of backing off exponentially.

[[thinking]]
Beyond the arithmetic bug, there's a design problem independent of it: this retries on every 5xx with no cap on total elapsed time and no jitter, so even a corrected exponential curve would still let many clients' backoff timers land in sync after a shared outage and hit the downstream in synchronized waves. I want to fix both the calculation and the shape, not just patch the off-by-one.

[[response]]
Rewrote the backoff to standard exponential-with-jitter, capped, and respecting `Retry-After` when the downstream sends one:

```go
func (c *Client) doWithRetry(ctx context.Context, req *http.Request) (*http.Response, error) {
    const (
        base   = 100 * time.Millisecond
        max    = 10 * time.Second
        maxAttempts = 6
    )
    for attempt := 0; attempt < maxAttempts; attempt++ {
        resp, err := c.http.Do(req)
        if err == nil && resp.StatusCode < 500 {
            return resp, nil
        }

        wait := backoffDelay(base, max, attempt)
        if err == nil {
            if ra := resp.Header.Get("Retry-After"); ra != "" {
                if secs, perr := strconv.Atoi(ra); perr == nil {
                    wait = time.Duration(secs) * time.Second
                }
            }
            resp.Body.Close()
        }

        select {
        case <-time.After(wait):
        case <-ctx.Done():
            return nil, ctx.Err()
        }
    }
    return nil, fmt.Errorf("exhausted %d retries", maxAttempts)
}

func backoffDelay(base, max time.Duration, attempt int) time.Duration {
    exp := base * time.Duration(1<<attempt)
    if exp > max {
        exp = max
    }
    // full jitter: uniform random in [0, exp)
    return time.Duration(rand.Int63n(int64(exp)))
}
```

Key changes: `attempt` drives `2^attempt` growth correctly (no off-by-one), full jitter avoids synchronized retry storms across clients, `Retry-After` is honored when present, capped at 10s so it can't grow unbounded, and it's now `ctx`-aware so a caller's own deadline cancels the wait instead of retrying past it.
